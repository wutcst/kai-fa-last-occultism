# Gensokoyo 技术文档

## 项目概述

本项目是一款基于 Unity 开发的东方风格纵向卷轴弹幕射击游戏，采用 C# 作为主要编程语言。游戏实现了完整的弹幕系统、敌人AI系统、道具系统以及多种视觉特效。

## 核心功能模块

### 1. 对象池 (Object Pooling)

对象池是本项目最重要的性能优化手段之一。在弹幕射击游戏中，子弹和特效对象的创建和销毁极为频繁，频繁的 `Instantiate` 和 `Destroy` 操作会造成严重的性能开销和内存碎片。

**实现方式：**

- 设计通用对象池类 `ObjectPooler`，支持预加载、动态扩容、对象回收
- 根据对象类型（子弹、特效、道具等）创建独立的对象池
- 使用队列（Queue）管理空闲对象，避免 GC 压力
- 对象回收时重置状态而非销毁重建

**应用场景：**

- 玩家子弹（激光、针状弹、符箓弹等）
- 敌机子弹（小米弹、大米弹、冰球、雪花等）
- 爆炸特效和击中特效
- 道具拾取物

***

### 2. 场景单例管理 (Scene Singleton Management)

游戏中存在多种跨场景存在的系统（如分数管理、音频管理、设置管理等），需要一种可靠的场景单例管理模式。

**实现方式：**

- 实现泛型单例基类 `Singleton<T>`，支持 DontDestroyOnLoad
- 使用 `lazy initialization` 延迟初始化，确保单例在首次访问时创建
- 线程安全的双重检查锁定（Double-Check Locking）模式
- 提供 `Instance` 属性统一访问入口

**管理的系统：**

- `GameManager` - 游戏主状态管理
- `ScoreManager` - 分数和成绩管理
- `AudioManager` - 音频播放管理
- `SettingManager` - 游戏设置管理

***

### 3. 状态机 (State Machine)

敌机AI和游戏流程需要复杂的状态切换逻辑。状态机模式提供了清晰、可扩展的状态管理方案。

**实现方式：**

- 设计接口 `IState` 定义状态的行为（Enter, Execute, Exit）
- 状态机类 `StateMachine` 管理当前状态和状态切换
- 每个具体状态实现 `IState` 接口
- 使用状态模式消除复杂的条件分支

**应用示例：**

- 敌人状态：巡逻、追击、攻击、逃跑、死亡
- BOSS状态：符卡前兆、符卡施放、符卡结算、形态转换、击败
- 玩家状态：正常、被击中、决死状态、无敌

***

### 4. 路径点贝塞尔曲线移动 (Bezier Curve Path Movement)

弹幕游戏中敌人的移动轨迹是视觉体验的重要组成部分。传统的直线运动过于单调，贝塞尔曲线提供了平滑且富有变化性的运动路径。

**实现方式：**

- 实现二次贝塞尔曲线和三次贝塞尔曲线算法
- 支持路径点数组定义复杂轨迹
- 使用参数 `t ∈ [0,1]` 控制沿路径的位置
- 支持闭合路径和开放路径

**数学公式（三次贝塞尔曲线）：**

```
B(t) = (1-t)?P? + 3(1-t)?tP? + 3(1-t)t?P? + t?P?
```

**应用场景：**

- 敌机出场动画轨迹
- BOSS移动路径
- 特殊弹幕的弧形轨迹
- 道具浮动轨迹

***

### 5. Perlin 噪声在视觉特效中的应用

Perlin噪声是一种连续的伪随机噪声函数，广泛用于程序化生成和自然现象模拟。

#### 5.1 雪花飘动效果

**实现方式：**

- 使用 Unity 的 `Mathf.PerlinNoise()` 函数
- 为每个雪花粒子计算基于时间的噪声偏移量
- 结合水平飘落和随机横向摆动
- 支持多层次噪声叠加增加自然感

**代码逻辑：**

```csharp
float noiseX = Mathf.PerlinNoise(time * speed, seed) * amplitude;
float noiseY = Time.time * fallSpeed;
```

#### 5.2 屏幕震动效果

**实现方式：**

- 基于Perlin噪声生成震动偏移量
- 支持衰减模式（屏幕震动随时间减弱）
- 可配置的震动强度和频率
- 支持不同震动模式（上下、左右、旋转）

**应用场景：**

- 玩家受击时的屏幕震动
- BOSS释放符卡时的震屏反馈
- 爆炸和大型特效的视觉增强

***

### 6. JSON 及 CSV 文件读取和解析

游戏数据（关卡配置、敌人数据、弹幕模式等）存储在外部文件中，运行时动态加载。

#### 6.1 JSON 数据处理

**实现方式：**

- 使用 Unity 内置 `JsonUtility` 类进行序列化/反序列化
- 设计数据类使用 `[Serializable]` 特性标记
- 支持嵌套对象和数组结构
- 实现异步加载避免阻塞主线程

**数据结构示例：**

```json
{
  "enemyData": [
    { "id": 1, "name": "妖精", "hp": 100, "speed": 2.0 },
    { "id": 2, "name": "魔法使", "hp": 300, "speed": 1.5 }
  ]
}
```

#### 6.2 CSV 数据处理

**实现方式：**

- 使用 `CsvHelper` 库或自定义解析器
- 支持逗号分隔和制表符分隔
- 自动跳过注释行和空行
- 类型转换和错误处理机制

**应用场景：**

- 关卡敌人配置表
- 弹幕模式数据表
- 成就解锁条件配置
- 成就排名数据

***

### 7. 分数本地保存 (Score Local Persistence)

玩家的游戏成绩需要持久化保存，用于高分榜展示和游戏进度记录。

**实现方式：**

- 使用 Unity 的 `PlayerPrefs` 存储简单数据
- 分数数据序列化为 JSON 后存储
- 支持本地高分榜（Top 10）
- 实现数据加密防止作弊

**存储结构：**

```json
{
  "topScores": [
    { "score": 50000, "date": "2026-05-14", "playerName": "Player1" },
    { "score": 45000, "date": "2026-05-13", "playerName": "Player2" }
  ],
  "totalPlayCount": 25,
  "totalPlayTime": 3600
}
```

## 性能优化点总结

| 优化项         | 优化效果                               |
| ----------- | ---------------------------------- |
| 对象池         | 减少 Instantiate/Destroy 调用，降低 GC 压力 |
| 场景单例        | 避免重复创建管理器，保证数据一致性                  |
| 状态机         | 代码结构清晰，减少条件分支，提高可维护性               |
| 贝塞尔曲线       | 预设路径计算，减少运行时轨迹计算                   |
| Perlin噪声    | GPU友好的程序化效果，减少美术资源                 |
| JSON/CSV配置  | 数据与代码分离，便于关卡设计和调整                  |
| PlayerPrefs | 轻量级本地存储，无需数据库支持                    |

***

## 开发注意事项

1. **对象池预加载**：游戏开始前预加载常用对象，避免运行时卡顿
2. **状态机切换**：状态退出时务必清理相关资源
3. **贝塞尔路径编辑**：提供可视化编辑器便于策划调整轨迹
4. **噪声参数调优**：Perlin噪声的频率和振幅需要根据视觉效果反复调整
5. **数据验证**：CSV/JSON加载后进行数据校验，防止崩溃
6. **分数安全**：重要分数数据建议加入校验和防止篡改
